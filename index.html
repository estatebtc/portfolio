Featured · Security · Chat
BlackIron — End-to-End Encrypted Chat

A privacy-first encrypted chat system with 13 layered security mechanisms. Messages are encrypted client-side with AES-GCM — the server only ever sees ciphertext. Identity is passwordless via Ed25519 keypairs. Session keys are derived using HKDF and rotate on demand. Forward secrecy, replay protection, signed commands, and TLS transport make this one of the most security-conscious hobby projects around.

AES-GCM E2EE
Client-side encryption. Server sees only ciphertext + metadata.
Ed25519 Auth
Passwordless. UUID derived from public key via SHA-256.
HKDF Session Keys
Per-session key derivation. One leak doesn't compromise others.
Forward Secrecy
/rekey generates new secret + salt, broadcast inside encrypted channel.
Replay Protection
Seq numbers enforced. Old packets and duplicates are rejected.
Signed Commands
Only owner can issue /kick /ban. Server verifies signature.
